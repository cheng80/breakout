# Swipe 벽돌깨기 제품 기획서

- 문서 버전: 2.0
- 작성일: 2026-07-21
- 기준: `Swipe-JS`, `SyphonArch/brick_breaker`, Swipe Brick Breaker 플레이 흐름
- 범위: 모바일 우선 5스테이지 싱글플레이 웹게임

## 1. 목표

하단을 한 손으로 스와이프해 발사 각도를 정하고 손을 떼면 여러 공을 일제 발사하는 벽돌깨기 게임을 만든다. 공은 벽과 벽돌을 튕기며 벽돌 내구도를 낮춘다. volley가 끝나면 남은 벽돌이 내려오고, 바닥에 닿으면 게임오버다. 모든 벽돌을 제거하면 다음 스테이지로 진행하며 5스테이지 완료 시 결과 화면을 표시한다.

이 문서는 이전 하트·적 공격·패들 추적형 게임 기준을 폐기하고 새 기준만 정의한다.

## 2. 기술 원칙

- TypeScript + Vite + PixiJS 기반으로 구현한다.
- 게임 규칙과 충돌은 렌더링과 분리한 순수 TypeScript로 둔다.
- 기존 의존성만 재사용하고 참고 레포지토리 코드는 복제하지 않는다.
- 입력은 Pointer Events를 우선하며 키보드 Space는 보조 발사 입력으로만 제공한다.
- 모바일과 데스크톱에서 동일한 논리 게임판을 사용하고 CSS로 화면에 맞춘다.

## 3. 핵심 플레이

### 조준과 발사

1. 시작 시 첫 공과 발사 위치는 하단 발사점에 붙어 있다.
2. 하단 조작 영역에서 pointerdown 후 드래그하면 조준선과 예상 방향을 표시한다.
3. pointerup/touchend에서 volley를 발사한다. pointercancel은 발사하지 않는다.
4. 한 volley의 모든 공이 회수되면 첫 공이 바닥에 닿은 x를 다음 발사 위치로 사용한다.
5. 다음 volley 전까지 공은 발사점에 붙어 있어야 한다.

### 벽돌과 진행

- 벽돌은 내구도 숫자를 표시한다.
- 공이 벽돌을 맞히면 내구도가 1 감소하고 0이면 제거된다.
- volley 종료 후 제거되지 않은 벽돌을 한 줄 하강시킨다.
- 벽돌이 하단 위험선에 닿으면 즉시 게임오버다.
- 모든 벽돌을 제거하면 즉시 다음 스테이지로 전환한다.
- 5번째 스테이지까지 완료하면 최종 결과 화면을 표시한다.

## 4. 아이템

| 아이템 | 규칙 | 표시 |
|---|---|---|
| 폭탄 | 획득 즉시 주변 3×3 벽돌에 1회 피해 후 소멸 | 폭탄 아이콘과 획득 피드백 |
| 멀티볼 | 다음 volley의 공 수를 1개 증가, 최대 8개 | 다음 volley 공 수 |
| 쉴드 | 바닥 도달 게임오버 1회를 막고 즉시 소모 | HUD 보호막 상태 |

아이템 효과는 현재 스테이지에서만 유지하고 스테이지 전환 시 초기화한다. 아이템 구분은 색상·모양·텍스트를 함께 사용한다.

## 5. 스테이지

최소 5개 스테이지를 제공한다. 스테이지마다 벽돌 배치와 내구도 패턴을 다르게 하고, 후반으로 갈수록 벽돌 수·내구도·하강 압박을 높인다. 단순 색상 변경만으로는 차별화로 인정하지 않는다.

## 6. 화면과 반응형 기준

- HUD: 스테이지, 점수, 남은 공 수, 아이템 상태, 게임 상태를 표시한다.
- 게임판: 벽돌, 공, 발사점, 조준선, 하단 위험선을 모두 표시한다.
- 발사 영역: 게임판 아래에 고정하고 패들과 겹치지 않게 한다.
- 좁은 모바일 화면에서도 패들·공·아이템·조작 영역이 잘리지 않아야 한다.
- 문서 스크롤 없이 `100vw`·`100dvh` 안에 맞춘다.
- 게임오버·승리 화면은 현재 점수와 재시작 버튼을 제공한다.

## 7. 수용 기준

1. 스와이프 조준과 pointerup volley 발사가 실제 브라우저에서 동작한다.
2. 공 충돌, 벽돌 내구도 감소, volley 회수, 첫 공 착지 위치 계승이 동작한다.
3. 벽돌 하강과 바닥 도달 게임오버가 동작한다.
4. 폭탄·멀티볼·쉴드가 서로 구분되고 규칙대로 적용된다.
5. 최소 5스테이지와 최종 승리·결과 화면이 동작한다.
6. 좁은 데스크톱·모바일 viewport에서 잘림과 문서 스크롤이 없다.
7. 핵심 규칙 자동 테스트, lint/typecheck/build와 브라우저 수동 검증이 통과한다.

## 8. 검증

- 자동: `npm test -- --run`, `npm run build`
- 수동: 데스크톱 viewport에서 조준→발사→벽돌 제거→게임오버·승리 흐름 확인
- 수동: 모바일 viewport에서 드래그·pointerup·반응형 레이아웃 확인
- 완료 전 실제 실행 URL과 변경 파일을 사용자에게 제시한다.

## 9. 제외 범위

계정, 랭킹, 광고, 서버 저장, 스테이지 선택, 이어하기, 온라인 동기화는 구현하지 않는다.
